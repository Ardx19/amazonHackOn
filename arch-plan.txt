## 1. Working Backwards — Press Release

**FOR IMMEDIATE RELEASE**

**Amazon Launches ReRoute: Every Returned Item Finds Its Next Best Owner**

*New AI system turns reverse logistics into a live marketplace, recovering maximum value at every point in the return journey*

Today, Amazon announces ReRoute — an intelligent resale platform embedded inside the Amazon app. When a customer returns an item, ReRoute's AI grades it in under 3 seconds and lists it as a discounted deal for nearby buyers while the item is still in transit. The discount is largest when the item is farthest from the warehouse — because that's where Amazon saves the most logistics cost by not shipping it further. As the item moves through hubs, the price rises automatically. Buyers who act fast get the deepest discount. Every item that sells in transit is a return that never reaches the warehouse.

ReRoute also enables a trusted C2C marketplace: customers can list used or outgrown items, get an AI-generated Product Health Card (immutable, Amazon-verified), and sell to a nearby buyer — with none of the friction or danger of OLX or Facebook Marketplace.

*"My daughter outgrew her baby monitor in two months. I wasn't going to deal with strangers. ReRoute let me list it in 30 seconds, and I had the money before lunch."*

## 2. The Problem

Amazon routes every return back to the source warehouse regardless of value — destroying ₹470 of a ₹500 item when logistics costs exceed recovery value. No intelligent system intercepts items mid-return, grades them instantly, and routes them to the highest-value exit.

---

## 3. Why v3 — The Stack Decision

The v2 architecture (Lambda + DynamoDB + Streamlit) was selected for maximum build speed under a solo night constraint. It succeeded — the AI grading pipeline and cascade routing engine are fully built and tested (7/7 tests pass).

The final stack optimizes for **demo impact and judging criteria**:

| Dimension | v2 (old) | v3 (final) | Why |
|---|---|---|---|
| Frontend | Streamlit | Next.js 14 App Router | SSR, mobile-first, professional UI. Judging panel sees a real web app, not a data dashboard |
| Backend | Lambda + API Gateway | FastAPI on App Runner | Warm container, async boto3, OpenAPI auto-docs. Judges can see the API surface |
| Database | DynamoDB | PostgreSQL 15 on RDS | Joins, FK constraints, migrations. Judges see relational integrity, not NoSQL convenience |
| AI Grading | Bedrock Claude 3.5 Sonnet | Bedrock Nova Lite | Speed + cost. Nova Lite is optimized for structured extraction; Claude is overkill for condition grading |
| AI Health Card | (not in v2) | Bedrock Claude 3.5 Sonnet | Prose quality matters for consumer trust. Claude writes better health assessments |
| QR Code | — | Python `qrcode` in FastAPI | Server-side PNG generation, no JS dependency |
| Deployment | Console zip | Amplify (frontend) + App Runner (backend) | GitHub push → deploy. Judges see CI/CD, not manual uploads |
| Cron | — | Lambda (kept) | Ring progression is fire-and-forget. Lambda is the right tool for scheduled tasks even with FastAPI |

**What stays**: All business logic (condition multipliers, cascade formulas, routing thresholds, ring price floors). All boto3 calls (Rekognition, Bedrock, S3). All seed data. Zero AI pipeline changes beyond the model ID.

## 4. Final Tech Stack

| Layer | Technology | Why |
|---|---|---|

| **Deployment** | Amplify (frontend) + App Runner (backend) | Both deploy from GitHub push |
| **Auth** | None (hardcoded personas) | Priya, Rahul, Ananya. Mention Cognito for production |
| **Cost** | < ₹1,500 for 48h | RDS t4g.micro ~₹300, App Runner ~₹500. Bedrock Nova Lite cheaper than Claude |

## 9. Core API Endpoints

### `POST /api/grade`
```
Input:  Multipart form — images[] (2-3 files) + item_id + original_price_inr + category
Output: GradingReport JSON — condition_grade, defects, confidence, estimated_retail_inr, suggested_resale_band_inr, recommended_route (preliminary)

AWS calls: Rekognition DetectLabels → Bedrock Nova Lite invoke_model
Writes: grading_reports row in PostgreSQL
```

### `POST /api/evaluate-route`
```
Input:  JSON — item_id + original_price_inr + category + current_location + ring_index
Output: RoutingDecision JSON — final_route, sale_price_inr, profitable_radius_km, listing_id, overhead_ratio, entered_reroute

Formula:  graded_value = multiplier × price → overhead_ratio → gate → cascade → ring price
Reads:    grading_reports (PostgreSQL)
Writes:   floating_discounts row (PostgreSQL)
```

### `POST /api/health-card`
```
Input:  JSON — item_id + seller_id + seller_name + seller_city
Output: HealthCard JSON — card_uuid, card_url, qr_code_base64, condition_grade, defects, confidence, immutable flag

AWS calls: Bedrock Claude 3.5 Sonnet invoke_model (prose generation) + qrcode lib (PNG → base64)
Reads:    grading_reports (PostgreSQL)
Writes:   health_cards row (PostgreSQL)
```

### `GET /api/deals`
```
Input:  Query params — hub_id (optional), max_radius (optional)
Output: list[FloatingDiscount] — filtered by hub and radius

Reads:   floating_discounts (PostgreSQL)
Used by: deals/page.tsx (marketplace grid)
```

## 10. PostgreSQL Schema

```sql
-- 7 tables, FK constraints, Alembic migrations

CREATE TABLE items (
    item_id TEXT PRIMARY KEY,
    name TEXT NOT NULL,
    category TEXT NOT NULL,
    brand TEXT,
    original_price_inr DECIMAL NOT NULL,
    image_filename TEXT,
    demo_condition_preset TEXT,
    demo_defects_preset JSONB,
    is_trajectory_product BOOLEAN DEFAULT false,
    trajectory_id TEXT
);

CREATE TABLE grading_reports (
    report_id UUID PRIMARY KEY,
    item_id TEXT NOT NULL REFERENCES items(item_id),
    product_category TEXT,
    brand_guess TEXT,
    condition_grade TEXT NOT NULL,
    defects JSONB DEFAULT '[]',
    completeness TEXT DEFAULT 'complete',
    confidence DECIMAL NOT NULL,
    estimated_retail_inr DECIMAL,
    suggested_resale_band_inr JSONB,
    recommended_route TEXT,
    routing_reason TEXT,
    manual_review_recommended BOOLEAN DEFAULT false,
    graded_at TIMESTAMP NOT NULL,
    rekognition_labels TEXT[] DEFAULT '{}'
);

CREATE TABLE floating_discounts (
    listing_id UUID PRIMARY KEY,
    item_id TEXT NOT NULL REFERENCES items(item_id),
    current_hub_id TEXT,
    ring_index INTEGER DEFAULT 0,
    original_price_inr DECIMAL NOT NULL,
    v_graded_inr DECIMAL NOT NULL,
    c_remaining_inr DECIMAL NOT NULL,
    mvsp_inr DECIMAL NOT NULL,
    current_sale_price_inr DECIMAL NOT NULL,
    discount_pct DECIMAL NOT NULL,
    radius_km DECIMAL NOT NULL,
    expires_at TIMESTAMP NOT NULL,
    status TEXT DEFAULT 'active',
    product_name TEXT,
    current_hub_name TEXT,
    trajectory JSONB
);

CREATE TABLE hub_checkpoints (
    checkpoint_id UUID PRIMARY KEY,
    item_id TEXT NOT NULL REFERENCES items(item_id),
    trajectory_id TEXT,
    hub_id TEXT NOT NULL,
    hub_name TEXT,
    arrived_at TIMESTAMP NOT NULL,
    c_remaining_inr DECIMAL NOT NULL,
    distance_to_warehouse_km DECIMAL NOT NULL
);

CREATE TABLE health_cards (
    card_uuid TEXT PRIMARY KEY,
    item_id TEXT NOT NULL REFERENCES items(item_id),
    card_url TEXT,
    condition_grade TEXT NOT NULL,
    defects JSONB DEFAULT '[]',
    brand_guess TEXT,
    product_category TEXT,
    confidence DECIMAL NOT NULL,
    seller_name TEXT,
    seller_city TEXT,
    amazon_guarantee BOOLEAN DEFAULT true,
    generated_at TIMESTAMP NOT NULL,
    grading_model_version TEXT
);

CREATE TABLE transactions (
    transaction_id UUID PRIMARY KEY,
    listing_id TEXT NOT NULL,
    buyer_id TEXT NOT NULL,
    seller_id TEXT NOT NULL,
    amount_inr DECIMAL NOT NULL,
    listing_type TEXT NOT NULL,
    status TEXT DEFAULT 'pending',
    created_at TIMESTAMP NOT NULL,
    completed_at TIMESTAMP
);

CREATE TABLE abuse_flags (
    account_id TEXT NOT NULL,
    rule_triggered TEXT NOT NULL,
    flagged_at TIMESTAMP NOT NULL,
    PRIMARY KEY (account_id, rule_triggered)
);
```

## 15. Risk Analysis

| Risk | Severity | Mitigation |
|---|---|---|
| Bedrock Nova Lite produces unreliable JSON | **CRITICAL** | Same `extract_json_from_response()` fallback as v2. Nova Lite is optimized for structured extraction — actually lower risk than Claude |
| Next.js 14 App Router learning curve | High | P3 starts with scaffold + layout before any business logic. Fetch API is universal — `lib/api.ts` wraps all backend calls |
| PostgreSQL schema drift from DynamoDB | Medium | Same logical schema. FK constraints are additive — they don't break existing logic. Alembic handles migrations |
| App Runner cold start | Low | Warm container. Unlike Lambda, no per-request cold start |
| QR code generation in FastAPI | Low | `qrcode` library is stable. Same pattern as v2's Python qrcode |
| Cron Lambda not triggering | Low | Manual trigger available in demo. Ring progression is a nice-to-have, not core flow |
